# 首页右侧栏布局优化说明文档

## 1. 背景与痛点
在首页（特别是右侧辅助信息栏）的显示中，存在以下两处核心布局问题：
- **快捷工具栏内部拥挤**：由于侧边栏宽度有限，原本的响应式断点（`lg={4}`）使得三个快捷工具在桌面端被压缩至极小宽度，导致文字强制竖向换行（如“政、策、搜、索”），图标与文字过度紧贴，影响阅读和点击。
- **最新政策 Banner 显示异常**：Banner 组件设计初衷可能为宽屏全通栏展示，内置了较高的固定高度（如 `280px`-`450px`）和较大的字号。将其放置在狭窄的右侧边栏时，会导致内部文字和按钮溢出、重叠，且上下元素层级关系混乱。

## 2. 优化改动点

### 2.1 快捷工具（QuickToolsSection）优化
- **网格布局调整**：将 `Col` 的响应式跨度统一修改为 `xs={8} sm={8} md={8} lg={8} xl={8}`。确保这3个工具按钮在任何容器宽度下，都能平均分配空间，始终保持同行并排显示。
- **卡片内部留白（Padding & Margin）**：
  - 增加卡片内部 `padding: "16px 8px"`，避免文字贴边。
  - 设置最小高度 `minHeight: "110px"`，保证卡片视觉体量一致。
- **排版与字号统一**：
  - 图标外层增加 `marginBottom: "12px"`，拉开与标题的间距。
  - 标题增加 `whiteSpace: "nowrap"` 防止换行错位，字号统一为 `14px`。
  - 描述文字限制为最多显示 2 行（`-webkit-line-clamp: 2`），行高调整为 `1.4`，确保多端阅读的舒适性。

### 2.2 轮播图（BannerSection）降级为侧栏模式
- **高度重置**：将原本针对宽屏的超大高度（280px~450px）统一下调至适合侧边栏的 `200px`，避免头重脚轻。
- **内边距与渐变遮罩**：遮罩层改为由下至上的渐变 `linear-gradient(180deg, rgba(0,0,0,0.1) 0%, rgba(0,0,0,0.8) 100%)`，文本对齐方式改为底部对齐（`align-items: flex-end`），避免遮挡核心图片内容。
- **字体与响应式清理**：
  - 移除了针对 `min-width: 1400px` 的大屏媒体查询，因为侧边栏容器宽度与屏幕物理宽度不直接等价。
  - 标题字号缩小至 `18px`，副标题缩小至 `13px`，并均增加多行截断处理（`line-clamp: 2`），防止文字在窄屏下挤占掉按钮的点击空间。
  - 按钮 padding 和字号适当减小，使其符合次级视觉权重。

## 3. 使用的设计 Tokens 及相对单位
- **间距（Spacing）**：采用 `8px` 倍数原则（如 8px, 12px, 16px, 24px），在卡片间距 `gutter={[16, 16]}` 及元素底部间距中应用。
- **字号（Typography）**：
  - 主标题：`18px` (Banner), `16px` (Card Title)
  - 正文/按钮：`14px` (Quick Tool Title), `13px` (Banner Subtitle)
  - 辅助说明：`12px` (Quick Tool Description)
- **圆角与边框（Border Radius）**：卡片与 Banner 统一使用 `8px` ~ `12px` 的柔和圆角，并附带 `30%` 透明度的主题色边框描边（如 `1px solid ${tool.color}30`）。

## 4. 后续维护建议
1. **容器查询（Container Queries）替代媒体查询**：若 Banner 等组件需要在全局多个不同宽度的区域复用，建议后续引入 CSS `@container` 特性，使组件能够根据“父容器宽度”而非“屏幕宽度”自适应。
2. **多语言与极端数据测试**：后续新增快捷工具时，确保描述文案不要超过 15 个汉字；若超出则依靠本次新增的 CSS 截断机制进行兜底。
3. **性能监控**：Banner 的多图轮播在低端设备上可能存在加载抖动，建议结合当前的 `SimpleErrorBoundary` 和 `Suspense` 补充图片懒加载 (Lazy Load) 策略。
