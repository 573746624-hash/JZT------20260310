# 申报管理模块需求方案

> 版本：v1.0  
> 创建时间：2026-03-23  
> 状态：需求设计阶段

---

## 一、项目背景与目标

### 1.1 背景
当前系统中"申报管理"与"我的申报"两个模块功能边界不清晰，用户体验存在以下问题：
- 两个模块入口分散，用户不清楚应该在哪里操作
- 申报流程不连贯，缺乏引导式的申报向导
- 申报进度不透明，用户难以掌握申报状态
- 缺乏企业级的申报统筹管理能力

### 1.2 目标
重新设计申报管理模块，实现：
1. **清晰的模块定位**：明确"申报管理"与"我的申报"的功能边界
2. **向导式申报流程**：提供步骤化的申报向导，降低申报门槛
3. **企业级管理能力**：支持从管理视角统筹企业申报事务
4. **个人化便捷体验**：支持从个人视角快速追踪和处理申报

### 2.2 模块关系说明

| 模块 | 定位 | 目标用户 | 核心功能 |
|------|------|----------|----------|
| **申报管理** | 企业级申报事务管理中心 | 企业管理员、财务人员、项目申报专员 | 统筹管理、数据分析、资源调配 |
| **我的申报** | 个人级申报事项追踪中心 | 企业内执行申报的员工 | 进度追踪、待办处理、历史查询 |
| **申报向导** | 申报流程核心引擎 | 所有申报人员 | 引导式申报、步骤化填写、材料上传 |

## 三、功能需求详述

### 3.1 申报管理（企业级）

#### 3.1.1 申报仪表盘

**功能描述**：提供企业申报数据的可视化概览

**页面元素**：
- **统计卡片区**：
  - 本月新增申报数
  - 进行中申报数
  - 待提交申报数
  - 审核中申报数
  - 已通过申报数
  - 需补充材料数

- **申报进度时间轴**：
  - 展示关键申报节点
  - 标注截止日期
  - 预警即将超期项目

- **快捷操作区**：
  - 新建申报按钮
  - 导入草稿按钮
  - 查看模板按钮

- **最近申报项目**：
  - 展示Top 5最近更新的申报
  - 显示项目名称、当前步骤、进度、截止日
  - 提供"继续申报"快捷入口

**交互逻辑**：
- 点击统计卡片可下钻到对应筛选状态的列表
- 点击申报项目进入申报向导对应步骤
- 点击"新建申报"启动申报向导

#### 3.1.2 申报列表（管理视图）

**功能描述**：企业全部申报项目的集中管理

**列表字段**：
| 字段 | 说明 | 排序 |
|------|------|------|
| 项目名称 | 申报政策/项目名称 | ✅ |
| 当前步骤 | 申报向导当前所在步骤 | ✅ |
| 申报状态 | 草稿/填写中/待提交/审核中/已通过/已驳回/需补充 | ✅ |
| 负责人 | 申报负责人姓名 | ✅ |
| 创建时间 | 申报创建时间 | ✅ |
| 截止时间 | 政策申报截止日期 | ✅ |
| 操作 | 查看/编辑/删除/指派/提醒 | - |

**筛选功能**：
- 政策类型筛选（多选）
- 申报状态筛选（多选）
- 时间范围筛选（创建时间/截止时间）
- 负责人筛选
- 关键词搜索

**批量操作**：
- 批量导出Excel
- 批量发送提醒
- 批量归档
- 批量删除（仅草稿状态）

**交互逻辑**：
- 点击"继续申报"进入申报向导对应步骤
- 点击"查看详情"弹出申报详情抽屉
- 支持列表/卡片视图切换

#### 3.1.3 数据统计分析

**功能描述**：提供企业申报数据的统计分析能力

**统计维度**：
- 申报数量趋势（按月/按季度/按年）
- 申报成功率分析
- 政策类型分布
- 申报状态分布
- 负责人工作量统计

**图表类型**：
- 折线图：申报趋势
- 饼图：类型分布
- 柱状图：状态分布
- 漏斗图：转化分析

### 3.2 申报向导（核心流程）

#### 3.2.1 向导流程设计

**步骤设计**：

| 步骤 | 名称 | 功能说明 | 预计耗时 |
|------|------|----------|----------|
| 步骤1 | **资质信息填写** | 填写企业基础信息、选择申报资质类型 | 10-15分钟 |
| 步骤2 | **证明材料上传** | 上传申报所需的各类证明文件 | 15-30分钟 |
| 步骤3 | **申报信息预览** | 预览完整的申报内容，检查完整性 | 5分钟 |
| 步骤4 | **确认提交** | 最终确认并提交申报 | 2分钟 |
| 步骤5 | **审核跟踪** | 提交后的审核状态跟踪 | 持续 |

**页面布局**：
```
┌─────────────────────────────────────────────────────────┐
│  步骤条                                                  │
│  ●─────●─────○─────○─────○                              │
│  资质   材料   预览   提交   跟踪                         │
│  填写   上传   确认   审核   查询                         │
├─────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────┐   │
│  │              当前步骤内容区域                    │   │
│  │                                                 │   │
│  │  （根据步骤显示不同的表单或上传组件）             │   │
│  │                                                 │   │
│  └─────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────┤
│  [上一步]          [保存草稿]          [下一步/提交]     │
└─────────────────────────────────────────────────────────┘
```

#### 3.2.2 步骤1：资质信息填写

**功能描述**：收集企业基础申报信息和资质选择

**表单内容**：
- **企业基础信息**（自动填充，可修改）：
  - 企业名称
  - 统一社会信用代码
  - 法定代表人
  - 注册资本
  - 成立日期
  - 所属行业
  - 企业规模
  - 注册地址
  - 联系方式

- **申报资质选择**：
  - 资质选择器（支持搜索、分类浏览）
  - 资质详情展示（申报条件、奖励政策、截止日期）
  - 智能推荐（基于企业画像匹配）
  - 多资质批量选择

- **联系人信息**：
  - 项目负责人
  - 联系电话
  - 电子邮箱
  - 项目团队（可选）

**功能特性**：
- 自动从企业画像填充基础信息
- 资质选择器支持智能搜索和筛选
- 实时校验资质申报条件匹配度
- 支持保存草稿

#### 3.2.3 步骤2：证明材料上传

**功能描述**：上传申报所需的各类证明材料

**材料清单**：
- 根据所选资质自动生成材料清单
- 区分必填材料和选填材料
- 标注材料要求和示例

**上传功能**：
- 支持拖拽上传
- 支持批量上传
- 文件格式校验（PDF、JPG、PNG等）
- 文件大小限制提示
- 图片预览功能
- 文件重命名
- 文件删除

**模板下载**：
- 提供材料模板下载
- 填写示例展示
- 常见问题说明

**上传状态**：
- 已上传/未上传标识
- 上传进度显示
- 上传成功/失败提示

#### 3.2.4 步骤3：申报信息预览

**功能描述**：预览完整的申报内容，进行最终检查

**预览内容**：
- 企业基础信息预览
- 申报资质列表
- 已上传材料清单
- 联系人信息

**完整性检查**：
- 自动检查必填项完整性
- 材料上传完整性检查
- 资质条件匹配度检查
- 风险提示（如：材料模糊、信息缺失等）

**操作按钮**：
- [返回修改] - 返回上一步修改
- [保存草稿] - 保存当前进度
- [确认提交] - 进入提交确认

#### 3.2.5 步骤4：确认提交

**功能描述**：最终确认并提交申报

**确认内容**：
- 申报信息最终确认
- 电子签章（如需要）
- 申报承诺书勾选
- 提交前风险提示

**提交操作**：
- 提交按钮二次确认
- 提交成功提示
- 生成申报编号
- 跳转至审核跟踪页面

#### 3.2.6 步骤5：审核跟踪

**功能描述**：提交后的审核状态实时跟踪

**跟踪内容**：
- 申报状态实时更新
- 审核进度步骤条
- 审核意见展示
- 补充材料提醒

**操作功能**：
- 查看申报详情
- 下载申报材料
- 撤回申报（审核前）
- 补充材料上传
- 联系审核人员

## 四、状态流转设计

### 4.1 申报状态定义

| 状态 | 说明 | 可操作 |
|------|------|--------|
| **草稿** | 已创建但未开始填写 | 编辑、删除 |
| **填写中** | 正在申报向导中填写 | 继续填写、保存草稿、删除 |
| **待提交** | 已填写完成但未提交 | 提交、编辑、保存草稿 |
| **审核中** | 已提交，正在审核 | 查看进度、撤回 |
| **需补充** | 审核反馈需要补充材料 | 补充材料、查看意见 |
| **已通过** | 审核通过 | 查看详情、下载证书 |
| **已驳回** | 审核未通过 | 查看原因、重新申报 |
| **已撤回** | 用户主动撤回 | 重新编辑、删除 |
| **已归档** | 已完成的申报归档 | 查看详情、恢复 |

### 4.2 状态流转图

## 五、交互设计规范

### 5.1 通用交互原则

1. **操作反馈**：所有操作均有明确反馈（Loading、Toast、动画）
2. **防错设计**：关键操作二次确认，删除操作需确认
3. **状态可见**：申报状态实时可见，状态变更及时通知
4. **进度保存**：支持随时保存草稿，防止数据丢失
5. **快捷导航**：支持快速跳转到相关页面

### 5.2 申报向导交互

1. **步骤导航**：
   - 支持点击已完成的步骤快速跳转
   - 未完成的步骤不可点击
   - 当前步骤高亮显示

2. **自动保存**：
   - 每5分钟自动保存草稿
   - 页面关闭前提示保存
   - 网络异常时本地缓存

3. **校验提示**：
   - 实时校验输入内容
   - 错误信息即时提示
   - 提交前完整性检查

### 5.3 列表交互

1. **筛选器**：
   - 支持多条件组合筛选
   - 筛选条件可清空
   - 筛选结果实时更新

2. **排序**：
   - 支持多字段排序
   - 排序状态可视化

3. **分页**：
   - 支持页码跳转
   - 支持每页条数设置
   - 支持快速翻页

## 六、数据需求

### 6.1 数据实体

#### 申报项目（Application）
```typescript
interface Application {
  id: string;                    // 申报ID
  projectNo: string;             // 申报编号
  name: string;                  // 项目名称
  policyId: string;              // 关联政策ID
  enterpriseId: string;          // 企业ID
  applicantId: string;           // 申请人ID
  applicantName: string;         // 申请人姓名
  
  // 状态信息
  status: ApplicationStatus;     // 申报状态
  currentStep: number;           // 当前步骤（1-5）
  progress: number;              // 进度百分比
  
  // 时间信息
  createTime: string;            // 创建时间
  updateTime: string;            // 更新时间
  submitTime?: string;           // 提交时间
  deadline: string;              // 截止日期
  
  // 内容信息
  basicInfo: BasicInfo;          // 基础信息
  qualifications: string[];      // 申报资质
  materials: Material[];         // 上传材料
  contactInfo: ContactInfo;      // 联系人信息
  
  // 审核信息
  auditLogs: AuditLog[];         // 审核记录
  auditOpinion?: string;         // 审核意见
  auditor?: string;              // 审核人
}
```

#### 申报材料（Material）
```typescript
interface Material {
  id: string;                    // 材料ID
  name: string;                  // 材料名称
  type: string;                  // 材料类型
  required: boolean;             // 是否必填
  fileUrl?: string;              // 文件地址
  fileName?: string;             // 文件名
  fileSize?: number;             // 文件大小
  uploadTime?: string;           // 上传时间
  status: 'pending' | 'uploaded' | 'verified' | 'rejected';  // 材料状态
}
```

## 七、非功能需求

### 7.1 性能需求

- 列表页加载时间 < 2秒
- 申报向导页面切换 < 1秒
- 文件上传支持断点续传
- 支持同时上传多个文件

### 7.2 安全需求

- 申报材料加密存储
- 敏感信息脱敏展示
- 操作日志记录
- 权限控制（数据权限+功能权限）

### 7.3 兼容性需求

- 支持Chrome、Firefox、Safari、Edge最新版本
- 支持移动端浏览器访问
- 响应式布局适配
