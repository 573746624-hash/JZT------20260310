# 璟智通 - 立即认证功能模块设计说明书

## 1. 模块架构设计
- **独立认证状态管理器 (`CertificationContext`)**
  统一维护用户的企业认证状态 (`status`, `level`, `certTime`, `expireTime`等)。
  它独立于普通的登录状态 (`AuthContext`)，并在全局顶层注入，供所有下游组件消费。
- **状态持久化**
  利用 `localStorage` 配合简单的Base64混淆加密 (`btoa/atob`)，实现刷新不丢失。同时记录了安全操作日志。

## 2. 功能实现亮点
- **未认证状态引导**：
  首页如果未认证，不仅会高亮提醒，还在点击核心路由时进行拦截，防止未认证用户操作。
- **多步骤认证流 (`EnterpriseCertificationModal`)**：
  采用了更严谨的"分步表单"设计：
  1. 上传营业执照 & 填写核心资料
  2. 法人代表活体/人脸识别（前端模拟交互）
  3. 认证成功 & 颁发企业专属邀请码
- **已认证状态管理 (`CompanyManagement`)**：
  在“企业管理”页面增设了独立的“实名认证信息”展示卡片。
  对敏感信息（身份证/统一信用代码、法人姓名）做了脱敏（Masking）处理。
  提供了“升级高级认证”和“过期重新认证”的续期入口。
- **路由级鉴权拦截 (`CertificationGuard`)**：
  除了按钮点击，还加入了 React Router 层面的 Guard 组件，包裹所有敏感的业务子路由（如 `/policy-center`, `/industry/service-match` 等），确保即使通过URL直达也会被拦截回认证提示页。

## 3. 技术实现与测试
- **React Context + Hooks**：使用了 `useCertification` 轻量级管理，不依赖沉重的 Redux，提高性能。
- **自动过期检测**：在 `CertificationContext` 初始化时自动判断 `expireTime`，一旦发现过期立即将状态回退为 `expired`。
- **单元测试覆盖**：
  采用 `Vitest` 和 `@testing-library/react` 编写了上下文状态转移的自动化测试，覆盖了认证提交、等级升级等核心流程（详见 `src/context/__tests__/CertificationContext.test.tsx`）。

## 4. 安全与合规
- 实现了敏感状态脱敏渲染（例：`$1**********$3`）。
- 提交过程设计了 `loading` 锁，避免防重复提交（防连点）。
- 控制台加入了基础的审计日志输出 (`[Audit Log]`) 机制，方便后续接入专业的监控上报。
