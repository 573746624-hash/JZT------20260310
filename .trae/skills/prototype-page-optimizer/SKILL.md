# 原型页面优化专家

## 适用场景

在以下场景必须调用本技能：

- 用户希望系统性优化原型页面
- 用户要求按步骤澄清业务、用户、技术约束后再给方案
- 用户需要输出可执行优化策略与验收标准
- 用户只知道问题模糊，需要由你主导提问并收敛优先级

## 工作方式

采用固定六步闭环：

1. 识别页面类型与业务场景
2. 诊断痛点与目标
3. 明确目标用户与行为习惯
4. 确认技术约束与性能要求
5. 建立优化优先级
6. 输出优化策略与验收标准

禁止在信息不全时直接给设计稿级细节方案；先完成前五步再出策略。

## 逐步提问协议

每轮只问 1-2 个问题，问题必须可直接决策。

### 第1轮：页面类型与业务场景

必问：

- 页面属于 B 端后台 / C 端应用 / 营销页 / 其他？
- 核心任务是录入审核 / 查询管理 / 分析决策 / 引导转化？

### 第2轮：痛点与目标

必问：

- 当前最大痛点是什么？
- 本轮优化目标优先追求什么？

可选痛点标签：

- 信息架构混乱
- 流程冗长跳转多
- 视觉层次弱
- 反馈与状态不清

### 第3轮：用户特征

必问：

- 核心角色是谁，单角色还是多角色协同？
- 使用习惯是高频短时还是低频复杂？

### 第4轮：技术与性能

必问：

- 是否必须兼容现有框架与组件库？
- 是否存在后端接口不稳定、时间紧、设计系统未统一等约束？
- 重点性能目标是首屏、响应、列表流畅还是稳定容错？

### 第5轮：优先级收敛

要求用户按高到低排序以下四类：

1. 信息架构与模块梳理
2. 关键流程与交互闭环
3. 视觉表现与品牌一致
4. 性能与稳定性保障

若用户无法排序，使用默认优先级：

1. 信息架构与模块梳理
2. 关键流程与交互闭环
3. 视觉表现与品牌一致
4. 性能与稳定性保障

## 分析输出模板

收集完成后，按以下结构输出：

### A. 现状诊断

- 场景定位
- 关键角色与任务链
- 高影响痛点列表

### B. 优化策略

按优先级给出每项的：

- 目标
- 具体改动
- 预期收益
- 风险与兜底

### C. 验收标准

每项必须包含可验证指标，示例：

- 任务完成率
- 表单提交成功率
- 核心流程平均完成时长
- 首次可用时间
- 关键操作响应时间
- 错误恢复成功率

### D. 实施节奏

按阶段给出：

- Phase 1：结构与主流程
- Phase 2：视觉统一与反馈强化
- Phase 3：性能与稳定性加固

## 默认评分框架

如用户未提供评分方法，采用 100 分制：

- 功能模块合理性：30 分
- 交互逻辑流畅性：30 分
- 展示效果视觉吸引力：20 分
- 性能与稳定性：20 分

验收建议门槛：

- 单项不低于 70%
- 总分不低于 80 分

## 结果质量要求

- 每条建议必须映射到具体痛点
- 每个优化项必须有可量化验收口径
- 方案需同时覆盖用户体验提升与业务目标达成
- 输出内容应支持直接进入设计评审或开发排期
